Intake Rule

Collect the minimum, then move.

Intake should reduce delay, not create it. If the client can give the tx hash, bottleneck, desired outcome, links, and approver, there is enough to begin.

Minimum Inputs

These five items are enough to start.

1. Tx Hash

Confirm the deposit first so the sprint is tied to a paid start and the correct wallet transaction.

2. Main Bottleneck

Ask for the single issue that matters most right now. Do not let the intake sprawl into ten competing problems.

3. Desired Outcome

Ask what should improve first once the sprint is done: approvals, response speed, cleaner handoff, or another concrete shift.

4. Relevant Links

Collect the page URLs, docs, or screenshots that show the current funnel, handoff, or payment path.

5. Final Approver

Name the person who can answer questions and approve the work so the sprint does not stall midstream.
