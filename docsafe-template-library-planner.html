Library Rule

Reusable document systems only scale when template composition, field portability, version naming, tenant handoff, and rollout QA are designed as one library.

Public repo issues keep pointing to the same library pressure: teams want bundles built from existing templates, exact field reuse across revised documents, visible submission naming for version control, prefilled download QA that actually reflects sent data, tenant-safe reply ownership, and template-level personalization that carries real client context into the invite.
