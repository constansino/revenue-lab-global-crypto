First-time wallet payment guide

If the buyer is ready but new to crypto payment, keep the instructions brutally simple.

This page is for first-time or low-confidence crypto payers. The goal is not to teach everything about wallets. The goal is to help them send the correct deposit on the correct network to the correct address, then share the tx hash so work can start.

Guide Rule

Fewer instructions close better than a full crypto lecture.

Buyers usually do not need a deep wallet tutorial. They need the minimum safe route: correct asset, correct network, correct address, and one confirmation step after sending.

Payment Map

Use the same simple sequence every time.

1. Confirm The Deposit Amount

Make sure the buyer knows the agreed deposit amount before they open the wallet. Do not leave the payment number vague.

Open Payment Page

2. Confirm The Network

Preferred route is USDC on Base. Fallback route is USDC or ETH on Ethereum. The network needs to match on both sides.

3. Copy The Exact Wallet

The buyer should copy the address directly and compare the first and last characters before sending.

4. Send The Payment

If they are unfamiliar, they can send a very small test first, then the main deposit. After the transfer, send the tx hash.

Common Mistakes

Most payment friction comes from a few repeated errors.

Wrong Network

Sending on an unsupported network creates avoidable recovery risk. Confirm Base or Ethereum before funds move.

Wrong Asset

Use the agreed asset only. Preferred is USDC for stable pricing, not a random token held in the wallet.

No Tx Hash Sent

Payment is not operationally complete until the tx hash is shared and kickoff can begin.

Short Payment Lines

Use concise lines when the buyer just needs the next step.

Network Line

Preferred route is USDC on Base. If you want Ethereum instead, confirm it first so the network matches on both sides.

Safety Line

Copy the wallet directly, check the first and last characters, and if you are unfamiliar, send a tiny test first.

Kickoff Line

After the payment lands, send the tx hash and the project brief. That is enough to move into kickoff.
