Post-win payment page

Once the next paid step is agreed, lock the next cycle cleanly.

This page is for the moment after a client accepts a second sprint or a first month of recurring work. The sale is not the problem anymore. The only job left is to confirm what this next payment covers, move the funds through the same wallet route, get the tx hash back, and activate the next cycle without losing momentum. If the current sprint is delivered but the remaining balance still has not been closed out, use balance collection first so the next cycle does not absorb unfinished money from the current one. If the next cycle is already approved but the payer still has not acted, use approval to payment before the thread softens into vague follow-up.
