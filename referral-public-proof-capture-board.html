What Is Usually Safe

Most referral-safe proof capture should stay inside these boundaries.

Safe To Reuse

The bottleneck, the type of fix, one short outcome line, and one anonymous description of the buyer pattern.

Sometimes Safe

A short quote, one sanitized delivery summary, or one bounded case angle with names and identifiers removed.

Usually Not Safe

Buyer names, referrer identity, screenshots, private economics, and the hidden referral operating path.

Best Default

Start with one short anonymous win summary unless the owner explicitly approves something wider.

Short Ask Lines

Use proof asks the owner can approve quickly.

Quote Ask

If useful, one short line on what changed and why this result felt worth repeating would already be enough.

Anonymous Ask

If naming is sensitive, I can keep this fully anonymous and only reference the team type, bottleneck, and visible shift.

Boundary Ask

The easiest version is to confirm what is safe to mention and what must stay hidden before anything gets reused publicly.

Forwardable Ask

I can turn this into one short line that supports the next aligned introduction without widening the story.
