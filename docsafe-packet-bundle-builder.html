Packet Rule

Multi-document workflows break when reusable templates, shared fields, signer roles, and change control are managed as separate afterthoughts.

Public repo issues keep pointing to the same packet pressure: teams want bundles that can include existing templates, shared roles and validation rules, fields copied between parties or templates, optional parties in parent-sign flows, and prepared data blocks reused across long form packets.
