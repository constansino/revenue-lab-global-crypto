Trust Router Rule

Promote one approved trust signal into one wider destination only when the smaller credibility block stops doing the job.

A trust asset can widen commercially without widening narratively. Good routing keeps the block compact, the page choice explicit, and the jump from trust to proof or case separate from story drift.

Best Destinations

Most widened trust signals belong in one of these six destinations.

Public Trust Page

Best when the asset still only needs one lightweight credibility job on one page.

Public Proof Page

Best when the signal needs one stronger public proof block but still does not need a fuller case narrative.

Public Proof Router

Best when the signal now supports a wider proof-layer decision and needs routing inside the proof system.

Public Case Router

Best when one trust signal is too small and the audience truly needs a broader before-after page.

Proof Library Slot

Best when the asset should stay reusable and compact before any broader public page placement happens.

Stay Small Instead

Best when broader public placement adds more exposure than leverage and the trust signal already clears doubt.
