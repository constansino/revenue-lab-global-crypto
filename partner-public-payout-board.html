1. Start Only After The Public Commercial Thread Is Real

Use this page only after the public-safe partner thread already has a real close, approval handoff, or payment motion. Payout logic should not be used to rescue a deal that is still vague.

2. Choose The Settlement Rule Before Funds Drift

Decide whether the thread is settling against one internal delivery rate, one direct client payment event, or one next-cycle renewal rule. Fix that logic before the money arrives.

3. Freeze What Must Stay Hidden During Settlement

Buyer identity, source path, raw screenshots, internal margin logic, and private operating mechanics stay off the public payout layer unless they are separately cleared.

4. Use One Proof Standard For Cleared Settlement

Once the amount and rule are fixed, use one simple proof standard: transfer sent, network confirmed, and tx hash returned in the same thread.

5. Move From Cleared Settlement Into Kickoff Or The Next Cycle

When the proof lands, stop discussing the same payout rule. Move straight into kickoff, renewal, or the next-cycle lock so the deal keeps momentum.
