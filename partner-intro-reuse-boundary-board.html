Warm intro proof rights page

Fix the reuse boundary before one approved proof line quietly turns into a broader public claim.

Warm-intro proof often becomes risky at the exact moment it becomes useful. A result line is approved, a quote sounds good, or a proof pack lands well, and then people start assuming the asset can be reused anywhere. This page prevents that drift. Freeze what is private, what is approval-only, what can help repeat intros, and what can travel publicly only in a narrow form.

Reuse Boundary Rule

Approval of the work is not the same as approval of future proof reuse.

Every proof asset should answer five things cleanly: what the asset is, who can see it now, where it can travel, what stays off-limits, and when the boundary needs to be reviewed again.

Reuse Classes

Most warm-intro proof should sit inside one of these tighter classes.

Private Only

The asset helps internal clarity but should not leave the current owner thread or packet.

Approval-Only

The asset can move inside an internal approval path but should still stay non-public and non-forwardable outside that lane.

Repeat-Lane Only

The asset can support similar repeat intros or pack forwards without becoming a broader website or case-study claim.

Partner-Safe Public

One anonymous or brand-light line is reusable publicly if buyer identity, economics, and the intro path remain protected.

Quote-Safe Public

One named or role-based line is approved, but only that line, not a wider testimonial package or narrative.

Next-Cycle Review

The asset may widen later, but only after a fresh review once the current cycle is closed and the next commercial step is separate.
