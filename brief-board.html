Project brief page

Make the brief short enough to send and strong enough to start.

This page gives a fixed structure for the post-payment project brief. The goal is not a long questionnaire. The goal is enough context to start the sprint without delay.

Brief Rule

One bottleneck is better than ten vague ideas.

The best brief names the current leak, the desired outcome, the relevant links, and any hard constraints. That is enough to start.

Brief Template

Use this exact structure.

1. Current Leak

What is the specific bottleneck right now?
Example:
leads can request a quote or make contact without getting a fast next step

2. Desired Outcome

What should improve first?
Example:
faster first response time, fewer missed quote opportunities, and cleaner intake details

3. Relevant Links

Send the main page URLs, docs, or screenshots that show the current funnel or handoff problem.

4. Constraints

Note any brand, compliance, timing, or tooling constraints that cannot be ignored during the sprint.

5. Owner

Name the person who will approve revisions or answer questions so the sprint does not stall.

6. Scope Reminder

Keep the first sprint narrow:
one follow-up message flow
one intake summary structure
one quote-prep or callback handoff
one owner-facing SOP

Project Brief

1. What is the main bottleneck you want fixed first?
2. What should improve when this sprint is done?
3. What page links, docs, or screenshots should I review?
4. Any hard constraints I should know up front?
5. Who is the final approver if questions come up?

Flow

Payment, then brief, then sprint.

This page sits after payment and before execution. Use it to reduce kickoff delay, not to add more process.
