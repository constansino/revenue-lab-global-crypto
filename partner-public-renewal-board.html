Public Renewal Rule

New sprint, new scope, new yes.

A public renewal page should answer five things fast: which next bottleneck matters, how narrow the new sprint stays, what the price anchor is, where the fresh deposit lands, and what stays outside the restart.

1. Start Only After Expansion Is Real

Use this page only when the first sprint is already closed and there is a clear next bottleneck worth packaging as a new sprint. If the next move is still fuzzy, stay in public expansion first.

2. Scope One Next Bottleneck Cleanly

3. Keep The Public Asset Attached To The Restart

4. Freeze What Stays Hidden During Renewal
