Transfer check page

Confirm the asset, network, and wallet before funds move.

Most stalled crypto payments are not pricing problems. They are last-minute operational doubts: what asset to send, which network to use, whether the wallet was copied correctly, and what to send back after payment. This page turns that into one visible checklist.

Network Checklist

Keep the transfer to four confirmations.

1. Confirm The Payment Type

Name whether this is a teardown deposit, sprint deposit, remaining balance, or first retainer month before any funds move.

Open Public Payment Type

2. Confirm The Asset And Network

Preferred route is USDC on Base. Only use another route if it was explicitly agreed in the same thread.

Open Public Payment Guide

3. Confirm The Exact Wallet

Copy the address directly and compare the visible start and end before sending.

4. Return The Tx Hash In The Same Thread

The transfer is only operationally complete when the tx hash is posted back into the same payment thread.
